# SAIL DESIGN PRINCIPLES

Founder:
Kalangiri Sainath

====================================

Every feature added to SAIL must satisfy these principles.

1. SIMPLE
Anyone should understand SAIL within one day.

2. READABLE
Code should read almost like English.

3. FAST
Both the compiler and programs should be fast.

4. SMART
The compiler should help programmers instead of only showing errors.

5. SAFE
Prevent common mistakes whenever possible.

6. MODERN
Built for AI, cloud and future software.

7. BEGINNER FRIENDLY
Learning SAIL should feel easier than learning Python.

8. PROFESSIONAL
Large companies should also be able to build products using SAIL.

9. CONSISTENT
One obvious way to do something.

10. UNIQUE
Never copy another language.
If another language has a better idea,
improve it before adding it.

====================================

QUESTION BEFORE ADDING ANY FEATURE

Does this make SAIL easier?

If NO

Don't add it.
